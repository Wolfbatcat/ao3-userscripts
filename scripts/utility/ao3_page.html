Installation

To use this skin on AO3, you'll be creating a small set of linked skins. The base file sets up variables and structure, but you'll also need to apply one of the theme files (Rose Pine Dawn, Rose Pine Moon, etc.) to actually get the colors.

Important: You must choose between the Normal Layout or the Card Layout. Once you pick a layout, use all the corresponding skins for that layout (base, tablet, and mobile). Do not mix skins from different layouts.

0. Choose Your Layout

Before you begin, decide which layout you prefer:

  • Normal Layout: Uses base.css, tablet.css, and mobile.css
  • Card Layout: Uses base_alt.css, tablet_alt.css, and mobile_alt.css

Check the previews above to see which style you prefer, then follow the instructions below using the files for your chosen layout. Note that both layouts use the same theme files, so you can pick any theme you like regardless of which layout you choose.

1. Create the Base Skin

  • Go to your Dashboard ‚Üí Skins ‚Üí Create Site Skin.
  • Give it a unique name, like [BBC] Ros√© Pine - Base, with "BBC" being your username or abbreviation.
  • Paste the contents of your chosen base file into the CSS field:
    • Normal Layout: base.css
    • Card Layout: base_alt.css
  • Under Advanced, set it to Parent Only.
  • Click Submit.

2. Create a Theme Skin

This is where the fun part happens‚Äîpicking your colors! Choose one of the themes below.

  • Create a skin titled something like [BBC] Ros√© Pine Dawn or [BBC] Ros√© Pine Moon.
  • Paste the code from one of the theme files:
    • Ros√© Pine Dawn
    • Ros√© Pine Moon
    • Ros√© Pine Moon Cool
    • Ros√© Pine
    • Ros√© Pine Cool
  • Set it to Parent Only and submit.
  • (Optional) Want a skin that switches between light and dark mode automatically based on your system preferences?
    • Create a second theme skin (e.g., [BBC] Ros√© Pine Moon for dark mode if you already created Dawn for light mode).
    • Paste in the CSS from your chosen second theme.
    • Set it to Parent Only.
    • Under Advanced ‚Üí Choose @media, select (prefers-color-scheme: dark) for your dark theme OR (prefers-color-scheme: light) for your light theme.
    • Go back to your first theme skin and edit it. Under Advanced ‚Üí Choose @media, select the opposite @media query ((prefers-color-scheme: light) if your second theme uses dark, or vice versa).
    • Now you have two themes that will switch automatically depending on your browser or device settings. When you get to Step 5, you'll add both theme skins to the parent chain.
    • Note: If you prefer to change your skins manually but want an easier way to do so then check out my AO3: Skin Switcher script. You can easily change site skins without leaving the page you were on.

2a. (Optional) Create an Alternative Tag Style Skin

Want a different tag style that's easier on the eyes with darker themes? This optional add-on changes how tags appear throughout the site.

  • Create a skin titled something like [BBC] Ros√© Pine - Tag Mod.
  • Paste the code from the tag mod file that matches your theme variant:
    • For regular Ros√© Pine themes (Dawn, Moon, or Ros√© Pine): mod_rosepine_tags.css
    • For Cool variants (Moon Cool or Ros√© Pine Cool): mod_rosepinecool_tags.css
  • Set it to Parent Only and submit.

3. Create the Tablet Skin

Note: The tablet skin is not optional and must be included for the theme to work properly across all screen sizes.

  • Create a skin titled [BBC] Ros√© Pine - Tablet.
  • Paste in the correct tablet file for your chosen layout:
    • Normal Layout: tablet.css
    • Card Layout: tablet_alt.css
  • Under Advanced ‚Üí Choose @media, select: only screen (max-width: 62em).
  • Set to Parent Only and submit.

4. Create the Mobile Skin

  • Create a skin titled [BBC] Ros√© Pine - Mobile.
  • Paste in the correct mobile file for your chosen layout:
    • Normal Layout: mobile.css
    • Card Layout: mobile_alt.css
  • Under Advanced ‚Üí Choose @media, select: only screen (max-width: 42em).
  • Set to Parent Only and submit.

5. Chain Them Together

  • Create one final skin, like [BBC] Ros√© Pine Dawn - Default (or [BBC] Ros√© Pine Auto - Default if you created two themes for auto-switching).
  • In the CSS field, paste .rose-pine {opacity: 1;} (just a placeholder so AO3 lets you save).
  • Under Advanced ‚Üí Parent Skins, add the skins you created in this order:
    1. [BBC] Ros√© Pine - Base
    2. Single theme: [BBC] Ros√© Pine Dawn / [BBC] Ros√© Pine Moon / [BBC] Ros√© Pine
      OR for auto light/dark switching: Add both of your theme skins (e.g., [BBC] Ros√© Pine Dawn and [BBC] Ros√© Pine Moon). The order doesn't matter since they use @media queries.
    3. (Optional) [BBC] Ros√© Pine - Tag Mod ‚Äî only if you created the alternative tag style skin in Step 2a
    4. [BBC] Ros√© Pine - Tablet
    5. [BBC] Ros√© Pine - Mobile
  • Click Submit, then Use.

Customization

AO3's code editor doesn't save annotations, so if you want to edit a skin, I highly recommend downloading it and using a proper code editor to make your changes.

Tip: For an easier time changing fonts and font sizes, check out the AO3: Site Wizard
userscript‚Äîit works on both PC and mobile.

  • Fonts: I recommend Merriweather (serif), Figtree (sans-serif), or Apfel Grotezk (sans-serif). If you use Apfel Grotezk, you can use Apfel Grotezk Fett (sans-serif) for headings. Also, Domine makes a great work/reader font‚Äîvery easy on the eyes. Code snippets use Victor Mono. If you want the cursive version of the code font, just remove the comments from /* font-style: italic; */ in your theme file.
    To change fonts: Open your theme skin (i.e. [BBC] Ros√© Pine Dawn) and press Ctrl+F to search for font-family. You'll see blocks like: 
    body,
  input,
  ...existing selectors...
  {
      font-family: Figtree, Bitter, Helvetica, ...;
  }
    Replace the font names in these font-family lines with your preferred fonts, keeping the fallback fonts for best compatibility. For example: 
    font-family: YourFont, Figtree, Bitter, Helvetica, Arial, sans-serif;
    Do the same for headings and code font sections if you want to change those as well.
  • Text Size: You can make text bigger or smaller using two variables in the theme skin: 
    /* TEXT SIZE */
--txt-size-main:       100%;     /* main text size */
--txt-size-work:       110%;     /* work text size */
    --txt-size-main controls text size across the whole site (navigation, blurbs, forms, etc.), while --txt-size-work is just for the actual work content. Bump up the percentage to make text larger (like 110% or 120%) or decrease it for smaller text (like 90%). Just be aware that going above 130% for --txt-size-main might cause some minor layout weirdness, though it'll still mostly work.
  • Tag Colors: Want to change up the tag colors? The variables are in the TAG COLORS section near the top of your theme file: 
    /*>========== TAG COLORS ==========<*/

/* BACKGROUND */
--tag-default-bg:      #9ccfd8;
--tag-fandom-bg:       #eb6f92;
--tag-warning-bg:      #f6c177;
--tag-ship-bg:         #ea9a97;
--tag-character-bg:    #3e8fb0;
--tag-freeform-bg:     var(--tag-default-bg);
--tag-splash-bg:       #F6ECE3;

/* TEXT */
--tag-default-txt:     var(--txt-alt1);
--tag-fandom-txt:      var(--txt-alt1);
--tag-warning-txt:     var(--txt-alt1);
--tag-ship-txt:        var(--txt-alt1);
--tag-character-txt:   var(--txt-alt1);
--tag-freeform-txt:    var(--txt-alt1);
    Just swap out the hex color values with whatever you like! For example, if you want purple fandom tags, change --tag-fandom-bg: #eb6f92; to --tag-fandom-bg: #c4a7e7;. You can also tweak the text color for each tag type using the --tag-*-txt variables.

    Note: Leave --tag-splash-bg alone‚Äîit's specifically for the splash page and doesn't affect regular tag colors.

    Important: If you're using the Alternative Tag Style add-on (Step 2a), you must change the tag colors in the tag add-on skin instead of the theme skin. The border and text colors are controlled by the tag text variables (like --tag-default-txt).
  • Platonic Ship Tags: Want platonic ships (relationships with the & symbol) to stand out with their own color? Add this rule to the bottom of your theme skin to give them a custom color: 
    li.relationships:has(a[href*="%20*a*%20"]) a.tag,
[class^="relationship"] a[href*="%20*a*%20"] {
  background-color: #c4a7e7 !important;
}
    Replace #c4a7e7 with any color you prefer. If you're using the tags add-on, make sure you add the code to the tags add-on instead of the theme skin and target the color and border properties instead of the background color.
  • Text Decorations: Those little symbols after tags and headers? You can change them! The rules are at the end of the theme CSS. For tags, it looks like this: 
    a.tag:not(.fandom.index.group a.tag,
h2.heading a.tag,
.splash .favorite a.tag,
h5.fandoms.heading a.tag,
dd.fandom.tags a,
[class^=warning] a.tag,
[class^=relationship] a,
[class^=character] a.tag):after {
  content: "\00a0\00a0‚úø";
}
    Swap out ‚úø for whatever symbol you want (‚ô•, ‚ú¶,  ö, etc.).
    • Important: Since the mobile skin uses the shortened version of the title "AO3 ‚úø", you'll have to change the symbol there too.

Add-Ons

These userscripts work great with the Ros√© Pine theme. Some can be customized to match the palette.

Note: If you're using a Chromium-based browser (Chrome, Brave, Vivaldi, etc.) or Microsoft Edge on PC, an extra step is needed to activate your userscript manager. Follow these instructions.

Mobile: Userscripts also work on mobile! On Android, use Firefox with the Tampermonkey add-on. On iPhone, use the Tampermonkey iOS app or the Stay App.

  • AO3: Advanced Blocker - Block works based on tags, authors, titles, word counts, and more. Filter by completion status, language, last update, and primary pairings / characters.
  • AO3: Skin Switcher - Change site skins from anywhere without leaving the page.
  • AO3: Site Wizard - Change fonts and font sizes from anywhere on the site. Customize how works appear with width, alignment, and spacing fixes.
  • AO3: Reading Time & Quality Score - Adds customizable reading time and engagement score bars to every work (available per chapter too!). To make the bars match this skin, open the settings menu under "Userscripts" on the top navigation bar and check off "Use icons instead of text labels" and optionally "Colored backgrounds" or "Colored text." It already uses Ros√© Pine's accent colors and icon colors by default!
  • AO3: Auto Pseud - Assign pseuds based on fandoms when commenting and bookmarking works.
  • AO3: Chapter Shortcuts - Adds a shortcut button to jump straight to the most recent chapter of any work. The shortcut symbol can match the text decoration symbol used throughout the theme.
  • AO3 Fictracker by Farin - An absolute lifesaver for tracking fics! Mark works as finished, favorite, or to-read using a customizable tag system. You can even sync across browsers and add custom, searchable notes to any fic. Here are my recommended settings:
    • Favorite - Rose rgb(235, 188, 186) for regular Ros√© Pine theme, or Iris rgb(196, 167, 231) for Cool variants. I change the tag and label to Reading.
    • To Read - Foam rgb(156, 207, 216)
    • Finished Reading - Subtle rgb(144, 140, 170). Cut the Opacity (not Border Opacity) in half to fade the work.
    • Subscribed - A custom tag I add to track my subscriptions‚Äîmakes them sortable with the bookmarks system. Use the same color as Favorite.
    • Set Border Size to 2 and adjust the Border Opacity to taste.
    • For the perfectionists out there: you can make the glowing border rounded to match the skin. To do this, open the script itself, find the StyleManager class/function, locate the line that says border-radius: 8px, and change 8px to 0.75em.
  • AO3 Floaty Comment Box (Responsive) by schildpath - Allows commenting on the fly, blends in with the site skin.
  • AO3: Comment Formatting and Preview by escctrl - Adds buttons to insert HTML formatting, and shows a live preview box of what the comment will look like.
